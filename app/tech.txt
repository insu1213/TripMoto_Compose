@구현 해야 하는 기능 (필수)
1. 지도 마커 Drag 후 Position 재 정의
2. Google Map의 Google로고를 가리지 않도록 표시하는 방안에 대해
Bottom Navigation을 선택적으로 보이는 기능 토글을 구현하는 방법.
Google Map 자체를 Padding을 주어 올리는 방법. (채택)

3. WishList나 지도에 사진을 표현하기 위해서는 Firebase Storage 사용이 필수적이지만,
LazyColumn을 ItemIndexed로 변경하였음에도 불구하고 Recomposition 시점을 고려하지 않으면 만들기 힘들 듯.
Recompositon을 적절히 관리하고 아니면, Screen이 표시되는 동안은 한번만 띄우는 것도 고려될 만 한듯.
아니면 일정 시간마다 업데이트 하도록 하여 필요할 때 사용자가 직접 Reload하도록 해야 할 듯.

4. Google Map 로딩 시간이나 Marker 편집 로딩 시간, WishList편집, 설정 로딩 시간에 모두 Loading 표시를 띄워야 함
5. Direction기능이 구현되지 않았음. 핵심 기능 중 하나인 Direction 기능을 구현하기 위해 여러 방법을 생각해야 함.
6. 비행기 티켓이나 숙소에 대한 정보 저장 스크린도 구현해야 함. 사용자가 일방적으로 적게 하는 방식이 현재 시점에서는 가장 적절할 듯.
7. 이중 저장에 대해 고려해야 함.
로컬과 Firebase Firestore에 이중 저장하여 오프라인 상태에서도 데이터에 접근하거나 수정할 수 있도록 해야 함. (수정은 못하게 막아도 됨)

8. 채팅 TextField 커스텀 해야 함.
